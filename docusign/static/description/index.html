E-SIGNATURE DOCUMENTS MANAGEMENT USING ODOO DOCUSIGN CONNECTOR

The Odoo Docusign Connector seamlessly integrates the capabilities of Odoo and Docusign, allowing businesses to manage and streamline their document signing processes directly within Odoo. This integration ensures enhanced efficiency, compliance, and an improved overall user experience. With features like real-time status tracking and secure storage, the connector eliminates manual bottlenecks and reduces errors. Businesses can send, track, and sign documents without leaving the Odoo platform, saving time and ensuring data consistency. Whether managing contracts, agreements, or approvals, the Odoo Docusign Connector empowers organizations to work smarter, faster, and with greater confidence.

Key Features and Functions

  • Digital Transaction Management
  • Fully integrated with Docusign for secure and efficient digital document handling
  • Send PDF documents (e.g., invoices, sales orders, purchase orders, or contacts) to recipients, including salespersons and customers, with a single button click. Supports multiple models like Contacts, Sales Orders, Purchase Orders and Invoices
  • Retrieve signed documents from recipients (salespersons or customers) with just one button click. Attach signed PDFs to their respective records
  • Send documents to a salesperson and to a customer in a seamless workflow, ensuring both parties sign the document. Each step is triggered with a single click
  • Monitor the current status of each document (e.g., sent, completed) at any time with a button click. Automatically update document status when the signature process is completed
  • Ensures only PDF documents are sent for signatures. Checks file extensions and ownership before processing
  • Automatically generate and attach PDFs (e.g., Quotation, Purchase Order) from Sales Orders and Purchase Orders to the Docusign Connector for signature

DOCUSIGN CREDENTIALS

  • You need Docusign Account (Demo or Live).

  • To generate an integrator and secret key login to your account, click the profile icon in the top right, and select Go To Admin.

  • After login or creating account go to settings.

  • In setting click on Apps and Keys. You can find this button on left side menu bar under Integration section.

  • In Apps and Keys create a new app and you can find your integration & secret key. If app already created then you just need to copy integration & secret key and save in odoo Docusign connector

  • if app already created then you just need to copy integration & secret key and save in odoo Docusign connector .

  • Now we need to install module "odoo_docusign" in Odoo.
    You will need to install 'docusign-esign' and 'httplib2' packages before installing our module. Following are the commands:
    1) pip install docusign-esign
    You can read here about it: https://pypi.org/project/docusign-esign/
    2) pip install httplib2
    You can read here about it: https://pypi.org/project/httplib2/
  • After installing the module, go to the Settings --> Users --> Your User .

  • Then provide Docusign credentials:
    • Name (Should be app name but it can be whatever you want)
    • Integrator Key
    • Secret Key
    • Account Type (Type of Docusign account; Demo or Live)
    • Account ID (Will be populated auto after "Get Access Token" steo)
  • Save Record.

  • Click on "Get login with Docusign account" to get login with your Docusign account.
  • Click on "Get Access Token" to get access token and account info.

SEND PDF DOCUMENTS FOR SIGNATURE

  • After getting access token, go to Contacts and click the button Get Signatures.

  • These functionalities are integrated in Contacts,Purchase Order,Invoices and Sale modules.
  • Attach a pdf document by clicking on Attachments (Only Pdf) . Ensure that the uploaded document is a pdf document.
  • Select a contact in Responsible as highlighted in the screenshot. Responsible is the one who is supervising this document signature.

  • You can add a recipient for this document signature. A recipient will receive the document via email and will be requested to review and sign the document
  • Once you have added the recipient, click on Send Document For Signatures button.

  • In the docs policy field there is a selection field to send the documents according to the current business need. there are two options available Simultaneously and In Hierarchy both serves the different purposes.
  • The In Hierarchy option is used to send the documents when there is a need for the signs of all the related recipients on a single documents in the order of their hierarchy set by the sender.Documents will be sent one by one, following the order of the recipients listed below.

  • With Simultaneously option Documents will be sent to all recipients at the same time.

  • Go to the recipient email as indicated in the screenshot and review the document
  • Once you have reviewed and signed it, come back in Odoo and click on update status button.
  • Now you can see that document's status is changed from Sent to Completed.

  • Click on Download Signed Document to download the signed document.
  • The same set of steps will be performed for Sales, Purchase Order and Invoicing

  • Click on Download Signed Document to download the signed document.
  • The same set of steps will be performed for Sales, Purchase Order and Invoicing
